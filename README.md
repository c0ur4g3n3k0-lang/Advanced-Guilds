# AdvancedGuilds

Высокопроизводительный плагин системы гильдий для **Purpur 1.21.11** (Paper API), написанный на **Java 21**. Плагин предоставляет полноценное управление гильдиями через GUI и команды: ранги, казна, дипломатические отношения, интеграция с Vault и PlaceholderAPI.

---

## Особенности архитектуры

| Компонент | Описание |
|-----------|----------|
| **База данных** | Асинхронная работа через **HikariCP** и `CompletableFuture`. Поддержка **SQLite** (по умолчанию) и **MySQL**. |
| **Кэш в памяти** | `ConcurrentHashMap` / `ConcurrentHashMap.newKeySet()` — потокобезопасный доступ без блокировок основного потока сервера. |
| **GUI** | Кастомные инвентари на базе `InventoryHolder` с обработчиками кликов через `GuildInventoryHolder`. |
| **Сообщения** | **Adventure API** + **MiniMessage** — градиенты, цвета, интерактивные элементы. |
| **Экономика** | Интеграция с **Vault** для создания гильдий, казны, депозитов и снятий. |
| **Плейсхолдеры** | Расширение **PlaceholderAPI** (`%guilds_*%`) для таба, чата и scoreboard. |

### Структура данных

- `GuildManager` — единая точка доступа к кэшу гильдий, приглашений, запросов на союз и уведомлений.
- `DatabaseManager` — DDL, загрузка и сохранение гильдий, участников, рангов и отношений.
- `GuildGuiFactory` — все игровые меню (создание, управление, ранги, участники, настройки).

## Команды

### Основные

| Команда | Аргументы | Право ранга | Описание |
|---------|-----------|-------------|----------|
| `/guild` | — | — | Открывает GUI: меню гильдии или создание (если игрок без гильдии). |
| `/guild create` | `<название>` | — | Создаёт гильдию (списание стоимости из Vault). Игрок не должен состоять в другой гильдии. |
| `/guild invite` | `<ник>` | `INVITE` | Отправляет приглашение игроку (действует 60 секунд). |
| `/guild join` | `<название>` | — | Принимает приглашение и вступает в гильдию с рангом минимального приоритета. |
| `/guild leave` | — | — | Покидает гильдию (лидер не может — только `/guild disband`). |
| `/guild kick` | `<ник>` | `KICK` | Исключает участника. Нельзя кикнуть игрока с равным или более высоким рангом. |
| `/guild chat` | `[сообщение]` | `CHAT_ACCESS` | Без аргумента — переключает режим гильдийского чата. С аргументом — отправляет сообщение. |
| `/guild info` | `[название]` | — | Показывает информацию о своей или указанной гильдии. |
| `/guild disband` | — | Владелец | Расформировывает гильдию (только лидер/владелец). |
| `/guild transfer` | `<ник>` | Владелец | Передаёт лидерство другому участнику. |
| `/guild pvp` | — | `MANAGE_SETTINGS` | Переключает дружественный огонь (friendly fire) между соклановцами. |

### Казна

| Команда | Аргументы | Право ранга | Описание |
|---------|-----------|-------------|----------|
| `/guild deposit` | `<сумма>` | — | Вносит личные средства в казну гильдии (Vault). |
| `/guild withdraw` | `<сумма>` | `BANK_WITHDRAW` | Снимает средства из казны на личный счёт. |

### Отношения

| Команда | Аргументы | Право ранга | Описание |
|---------|-----------|-------------|----------|
| `/guild ally` | `<гильдия>` | `MANAGE_SETTINGS` | Отправляет запрос на союз (таймаут 60 сек). |
| `/guild allyaccept` | `<гильдия>` | `MANAGE_SETTINGS` | Принимает входящий запрос на союз. |
| `/guild enemy` | `<гильдия>` | `MANAGE_SETTINGS` | Объявляет войну (односторонне). Разрывает союз, если он был. |
| `/guild neutral` | `<гильдия>` | `MANAGE_SETTINGS` | Сбрасывает отношения до нейтральных (союз и война). |

> При изменении отношений все **онлайн-участники обеих гильдий** получают уведомление через Adventure MiniMessage.

### Административные (`advancedguilds.admin`)

| Команда | Описание |
|---------|----------|
| `/guilds delete <название>` | Принудительно удаляет гильдию. |
| `/guilds info <название>` | Подробная информация для администратора. |
| `/guilds setbalance <название> <сумма>` | Устанавливает баланс казны. |

## Права рангов (GuildPermission)

Настраиваются в `config.yml` → `default-ranks` и через GUI «Управление рангами»:

| Право | Описание |
|-------|----------|
| `INVITE` | Приглашение игроков |
| `KICK` | Исключение участников, управление рангами (с проверкой приоритета) |
| `MANAGE_SETTINGS` | Настройки, PvP, дипломатия, смена рангов |
| `BANK_WITHDRAW` | Снятие из казны |
| `CHAT_ACCESS` | Доступ к гильдийскому чату |

Приоритет ранга определяет иерархию: игрок не может изменять ранг или кикать участника с **равным или более высоким** приоритетом.

## PlaceholderAPI

Идентификатор расширения: `guilds`

| Плейсхолдер | Описание | Без гильдии |
|-------------|----------|-------------|
| `%guilds_name%` | Название гильдии | `""` |
| `%guilds_tag%` | Цветной тег (MiniMessage → Legacy) | **`""`** (пустая строка, без пробелов) |
| `%guilds_rank%` | Отображаемое имя ранга (`displayName`) | Значение из `placeholders.no-rank` (по умолчанию `Нет`) |
| `%guilds_balance%` | Баланс казны (формат Vault) | `""` |
| `%guilds_members_online%` | Онлайн-участники | `0` |
| `%guilds_members_total%` | Всего участников | `0` |

> `%guilds_tag%` специально возвращает пустую строку для игроков без гильдии, чтобы не ломать форматирование чата и таба.

## GUI

- **Главное меню** — список участников, информация о гильдии (союзники, враги, нейтральные гильдии), настройки, выход.
- **Управление участником** — повышение/понижение ранга, исключение (с проверкой приоритета).
- **Настройки** — тег, цвет тега, friendly fire, ранги, смена названия.
- **Ранги** — редактирование прав каждого ранга.

В блоке информации отображаются:
- **Союзники** — цветные теги из `guild.allies`
- **Враги** — цветные теги из `guild.enemies`
- **Нейтралитет** — до 5 тегов остальных гильдий + «и ещё X...»

## Пример конфигурации

```yaml
database:
  mysql: false
  host: "localhost"
  port: 3306
  name: "advancedguilds"
  user: "root"
  password: "password"
  pool-size: 10

guild:
  creation-cost: 5000.0
  max-members: 50
  name:
    min-length: 3
    max-length: 16
    regex: "^[a-zA-Z0-9а-яА-Яa-zA-Z_]+$"
  tag:
    min-length: 2
    max-length: 5
    regex: "^[a-zA-Z0-9а-яА-Я_]+$"
  chat:
    format: "<dark_gray>[<gradient:gold:yellow>Гильдия</gradient>]</dark_gray> <gray>[<rank>]</gray> <white><player></white><dark_gray>:</dark_gray> <yellow><message></yellow>"

default-ranks:
  - key: "leader"
    name: "Лидер"
    priority: 100
    permissions:
      - INVITE
      - KICK
      - MANAGE_SETTINGS
      - BANK_WITHDRAW
      - CHAT_ACCESS
  - key: "officer"
    name: "Офицер"
    priority: 75
    permissions:
      - INVITE
      - KICK
      - CHAT_ACCESS
  - key: "veteran"
    name: "Ветеран"
    priority: 25
    permissions:
      - CHAT_ACCESS
  - key: "recruit"
    name: "Рекрут"
    priority: 1
    permissions:
      - CHAT_ACCESS

messages:
  prefix: "<dark_gray>[<gradient:gold:yellow>Гильдии</gradient>]</dark_gray> "
  relation-ally-established: "<green>Наша гильдия заключила союз с гильдией <tag>!</green>"
  relation-ally-broken: "<red>Союз с гильдией <tag> был расторгнут!</red>"
  relation-enemy-declared: "<dark_red>Гильдия <tag> объявила нам войну!</dark_red>"
  relation-enemy-declared-source: "<dark_red>Наша гильдия объявила войну гильдии <tag>!</dark_red>"
  relation-neutral-set: "<yellow>Отношения с гильдией <tag> сброшены до нейтральных.</yellow>"
  gui-relations-allies: "<gray>Союзники:</gray>"
  gui-relations-enemies: "<gray>Враги:</gray>"
  gui-relations-neutral: "<gray>Нейтралитет:</gray>"
  gui-relations-none: "<gray>Нет</gray>"
  gui-relations-more: "<gray> и ещё <white><count></white>...</gray>"
  friendly-fire-on: "<green>Дружественный огонь включён.</green>"
  friendly-fire-off: "<red>Дружественный огонь выключен.</red>"

placeholders:
  no-rank: "Нет"
```

Полный список сообщений — в файле `src/main/resources/config.yml`.
